# ReviewInbox

ReviewInbox turns public app store reviews into reply workflows and recurring product pain signals for mobile app teams.

## Language

**ReviewInbox**:
The product name for the open source and hosted service at `github.com/bibliocards/reviewinbox` and `reviewinbox.app`.
_Avoid_: Review Pain Digest, Pain Digest

**Open Source Core**:
The self-hostable product that remains useful on its own, including store review ingestion, reply drafting, human approval, publishing, and weekly digests.
_Avoid_: Community teaser, crippled edition

**Cloud Service**:
The hosted ReviewInbox offering that sells convenience around setup, scheduling, delivery, backups, limits, and team operations.
_Avoid_: Separate product, enterprise-only version

**Deployment Mode**:
The runtime configuration that tells the same ReviewInbox codebase whether it is running as self-hosted software or as the hosted cloud service.
_Avoid_: Edition, fork, distribution

**Organization**:
The tenant boundary for users, apps, store connections, reviews, reply workflows, and digests. Self-hosted installations may start with one default organization.
_Avoid_: Workspace, account, team

**Owner**:
The first user of an organization, responsible for setup and administration.
_Avoid_: Admin, superuser

**App**:
A mobile product tracked by ReviewInbox. An app can have one Apple App Store connection, one Google Play connection, or both.
_Avoid_: Project, product

**Store Connection**:
The configured link between an app and an external review source such as Apple App Store or Google Play.
_Avoid_: Integration, connector, credential

**Store Credential**:
A secret used by a store connection to read reviews or publish replies through an external store API.
_Avoid_: Token, key, integration config

**Reply Inbox**:
The primary ReviewInbox workflow where new store reviews are reviewed, drafted, edited, approved, ignored, or published back to the stores.
_Avoid_: Feedback board, support inbox

**Review**:
A user review imported from an external store for a specific app and store connection.
_Avoid_: Feedback, message, comment

**Reply Draft**:
A proposed store response for a review, generated by AI or edited by a user before publication.
_Avoid_: Suggested answer, response draft

**Reply Language Policy**:
An app-level rule that decides which language ReviewInbox should use when drafting replies.
_Avoid_: Translation settings, locale config

**Reply Context**:
App-level Markdown guidance injected into AI reply drafting when relevant, such as known incidents, support policies, or version-specific fixes.
_Avoid_: Knowledge base, prompt notes, system prompt

**Published Reply**:
A reply that ReviewInbox has successfully published back to the external store.
_Avoid_: Response, answer

**Sync Run**:
A recorded attempt to fetch reviews from a store connection.
_Avoid_: Import, crawl, job

**Weekly Digest**:
A recurring Markdown or email summary of product pain patterns found in reviews.
_Avoid_: Dashboard, report

**Managed AI**:
The AI generation capacity provided by the ReviewInbox cloud service without requiring customers to bring their own provider key.
_Avoid_: Built-in OpenAI, free AI

**Bring Your Own Key**:
An AI configuration where an organization provides its own provider credentials for OpenAI, Anthropic, Ollama, or another OpenAI-compatible service.
_Avoid_: Custom AI, external AI

**Usage Event**:
A recorded billable or limit-relevant action, such as importing a review, generating a reply draft, publishing a reply, or generating a digest.
_Avoid_: Metric, analytics event
